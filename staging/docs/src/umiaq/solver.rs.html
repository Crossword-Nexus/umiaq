1//! The main solver for pattern-matching equations against entry lists.
2//!
3//! # Error Handling
4//!
5//! The solver uses [`SolverError`] with three variants:
6//!
7//! - S001: `ParseFailure` (Pattern parsing failed (wraps [`ParseError`]))
8//! - S002: `NoPatterns` (Equation has only constraints, no patterns to solve)
9//! - S003: `MaterializationError` (Internal error during solution construction)
10//!
11//! Each error has a `code()`, optional `help()`, and `display_detailed()` method.
12//!
13//! # Examples
14//!
15//! ## Basic Usage
16//!
17//! ```
18//! use umiaq::solver;
19//!
20//! let entries = vec!["cat", "dog", "catalog", "dogma"];
21//! let result = solver::solve_equation("A*B", &entries, 10)?;
22//!
23//! println!("Found {} solutions", result.solutions.len());
24//! for solution in result.solutions {
25//!     println!("{}", solver::solution_to_string(&solution)?);
26//! }
27//! # Ok::<(), Box<dyn std::error::Error>>(())
28//! ```
29//!
30//! ## Handling Errors with Detailed Messages
31//!
32//! ```
33//! use umiaq::solver::{self, SolverError};
34//!
35//! let entries = vec!["test"];
36//! match solver::solve_equation("", &entries, 10) {
37//!     Ok(result) => println!("Success: {} solutions", result.solutions.len()),
38//!     Err(e) => {
39//!         // Show detailed error with code and help
40//!         eprintln!("{}", e.display_detailed());
41//!         // Error code: S001
42//!         // Help text: Example: Use 'A*B' or '*cat*' instead of empty string
43//!     }
44//! }
45//! ```
46//!
47//! ## Checking Solve Status
48//!
49//! ```
50//! use umiaq::solver::{self, SolveStatus};
51//!
52//! let entries = vec!["cat", "dog"];
53//! let result = solver::solve_equation("A*", &entries, 100)?;
54//!
55//! match result.status {
56//!     SolveStatus::FoundEnough => println!("Found all requested results"),
57//!     SolveStatus::EntryListExhausted => println!("Searched entire entry list"),
58//!     SolveStatus::TimedOut { elapsed } => {
59//!         println!("Timed out after {:?}", elapsed);
60//!     }
61//! }
62//! # Ok::<(), Box<dyn std::error::Error>>(())
63//! ```
64//!
65//! # Performance Characteristics
66//!
67//! ## Time Complexity
68//!
69//! The solver's performance depends heavily on pattern structure and entry-list size:
70//!
71//! - **Best case**: O(n × k) where n = entry-list size, k = max bucket size
72//!   - achieved when patterns share many variables (good bucketing)
73//!   - example: `AB;BA;BC` with 10,000 entries → ~10,000 iterations
74//!
75//! - **Worst case**: O(n^p) where n = entry-list size, p = number of patterns
76//!   - occurs when patterns share no variables (no bucketing)
77//!   - example: `A;B;C` with 10,000 entries → 10,000³ = 1 trillion iterations
78//!
79//! - **Middle-ground case**: O(n × k^p) where k ≪ n
80//!   - Moderate variable sharing provides some bucketing benefit
81//!
82//! ## Key Performance Factors
83//!
84//! ### 1. Pattern Ordering
85//!
86//! Pattern ordering potentially has an exponential impact. The `get_ordered_patterns` heuristic
87//! (see patterns.rs) maximizes shared variables between consecutive patterns:
88//!
89//! ```text
90//! Good:  ABC → AB → A  (2 shared, then 1 shared)
91//! Bad:   A → B → C     (0 shared, 0 shared)
92//! ```
93//!
94//! See `patterns::get_ordered_patterns` for details on the ordering algorithm.
95//!
96//! ### 2. Constraint Propagation
97//!
98//! Joint constraints like `|ABC| = 10` tighten variable bounds before search:
99//!
100//! ```text
101//! Before: A,B,C ∈ [1,∞) → search space = "∞"³
102//! After:  A,B,C ∈ [1,8]  → search space = 8³ = 512
103//! ```
104//!
105//! See `joint_constraints::propagate_joint_to_var_bounds` for implementation.
106//!
107//! ### 3. Adaptive Batch Sizing (Time/Memory Trade-off)
108//!
109//! The solver scans the entry list in adaptive batches (10k → 20k → 40k...):
110//!
111//! - **Small batches**: lower memory, more join iterations
112//! - **Large batches**: higher memory, fewer join iterations
113//! - **Adaptive**: balances memory usage with search efficiency
114//!
115//! ### 4. Time Budget (Robustness)
116//!
117//! Timeout (currently 30s) prevents queries from running indefinitely:
118//!
119//! - checks budget between candidates in recursive join
120//! - returns partial results or error when budget expires
121//! - prevents exponential blowup from consuming unlimited resources
122//!
123//! ## Expensive Equations
124//!
125//! Equations potentially become computationally expensive when:
126//!
127//! 1. **Many patterns** (e.g., 5+): potentially exponential with number of patterns
128//! 2. **No shared variables**: each pattern multiplies search space
129//! 3. **Unconstrained variables**: variables with bounds [1,∞) are costly
130//! 4. **Large entry lists**: linear factor, but multiplied by pattern count
131//!
132//! Example expensive queries:
133//! ```text
134//! A;B;C;D;E    → 5 patterns, no sharing (exponential)
135//! ABCDEFGH     → 8 vars, no joint constraints (huge search space)
136//! ```
137//!
138//! ## Optimization Strategies
139//!
140//! The solver uses several optimizations to handle complex queries:
141//!
142//! 1. **Prefilter regex**: rejects nonmatching entries before detailed matching
143//! 2. **Deterministic fast path**: materializes fully determined patterns directly
144//! 3. **Early termination**: stops when quota met or budget expires
145//! 4. **Deduplication**: hash-based set prevents duplicate solutions
146//! 5. **String interning**: reduces allocation overhead via Rust's Rc<str>
147//!
148//! ## Memory Usage
149//!
150//! Peak memory usage is dominated by:
151//!
152//! - **Candidate buckets**: O(n × patterns) (where n = entry-list size)
153//! - **Solutions**: O(#results × patterns)
154//! - **Dedup set**: O(s), where s = # of unique solutions
155//!
156//! Typical usage: 10-100 MB with a 100k entry list.

236/// Unified error type for the solver pipeline.
237///
238/// This consolidates the different error sources we encounter when parsing
239/// equations, materializing solutions, or respecting time budgets, so that
240/// callers only need to handle a single `Result<_, SolverError>`.
241#[derive(Debug, thiserror::Error)]
242pub enum SolverError {
243    /// Failure during parsing of the equation string into an `EquationContext`.
244    ///
245    /// These originate from the parser (`ParseError`), which we box to keep the
246    /// error type size stable.
247    #[error("parse failure: {0}")]
248    ParseFailure(#[from] Box<ParseError>),
249
250    /// The equation contained only constraints and no patterns,
251    /// so there is nothing to solve.
252    #[error("no patterns to solve (constraints only)")]
253    NoPatterns,
254
255    /// Failure while materializing a candidate binding into a concrete solution.
256    ///
257    /// This generally indicates that an internal constraint check failed during
258    /// `recursive_join` or related routines.
259    #[error("materialization error: {context}")]
260    MaterializationError { context: String },
261}
262
263impl SolverError {
264    /// Returns the error code for this error variant
265    #[must_use]
266    pub fn code(&self) -> &'static str {
267        match self {
268            SolverError::ParseFailure(_) => "S001",
269            SolverError::NoPatterns => "S002",
270            SolverError::MaterializationError { .. } => "S003",
271        }
272    }
273
274    /// Returns a short description of this error type (for documentation)
275    #[must_use]
276    pub fn description(&self) -> &'static str {
277        match self {
278            SolverError::ParseFailure(_) => "Pattern parsing failed",
279            SolverError::NoPatterns => "Equation has only constraints, no patterns to solve",
280            SolverError::MaterializationError { .. } => "Internal error during solution construction",
281        }
282    }
283
284    /// Returns detailed explanation of this error type (for documentation)
285    #[must_use]
286    pub fn details(&self) -> &'static str {
287        match self {
288            SolverError::ParseFailure(_) => "The input pattern could not be parsed. This wraps an underlying ParseError (see Parse Errors section for specific error codes).",
289            SolverError::NoPatterns => "The equation contains only constraints (like `|A|=3`) but no actual patterns to match against entries. Add at least one pattern like `A*B` or `*cat*`.",
290            SolverError::MaterializationError { .. } => "This indicates an internal solver error where a pattern matched but constraints could not be satisfied during solution materialization. This is usually a bug in the solver logic.",
291        }
292    }
293
294    /// Returns a helpful suggestion for this error
295    #[must_use]
296    pub fn help(&self) -> Option<&'static str> {
297        match self {
298            SolverError::NoPatterns => Some("Add at least one pattern to solve. Example: 'A*B' or '*cat*;*dog*'"),
299            SolverError::MaterializationError { .. } => Some("This is an internal error. The pattern matched but constraints could not be satisfied."),
300            SolverError::ParseFailure(_) => None, // ParseError has its own help
301        }
302    }
303
304    /// Formats the error with code and optional help text
305    #[must_use]
306    pub fn display_detailed(&self) -> String {
307        match self {
308            SolverError::ParseFailure(pe) => {
309                // delegate to ParseError's detailed display
310                format!("{}\n  caused by: {}", self.code(), pe.display_detailed())
311            }
312            _ => {
313                crate::errors::format_error_with_code_and_help(&self.to_string(), self.code(), self.help())
314            }
315        }
316    }
317}

326/// Bucket key for indexing candidates by the subset of variables that must agree.
327/// - `None` means "no lookup constraints for this pattern" (Python's `entries[i][None]`).
328/// - When present, we store a *sorted* `(var_char, var_val)` list so the key is deterministic
329///   and implements `Eq`/`Hash` naturally. This mirrors Python's
330///   `frozenset(dict(...).items())`, but with a stable order.
331/// - The sort happens once when we construct the key, not on hash/compare.
332pub type LookupKey = Vec<(char, Rc<str>)>;
333
334/// Result of attempting to build a lookup key from a binding
335enum LookupKeyResult {
336    /// successfully built the lookup key (empty if no keys needed)
337    Complete(LookupKey),
338    /// a required key variable was not found in the binding
339    MissingKey,
340}

352/// All candidates for one pattern ("bucketed" by `LookupKey`).
353/// - `buckets`: groups candidate bindings that share the same values for the
354///   pattern's `lookup_keys` (variables that must align with previously chosen patterns).
355/// - `count`: mirrors Python's `word_counts[i]` and is used to stop early when a global cap
356///   per-pattern is reached (e.g., `MAX_ENTRY_COUNT`). We track it here to avoid recomputing.
357#[derive(Debug, Default)]
358#[derive(Clone)]
359pub struct CandidateBuckets {
360    /// Mapping from lookup key -> all bindings that fit that key
361    pub buckets: HashMap<LookupKey, Vec<Bindings>>,
362    /// Total number of bindings added for this pattern (across all keys)
363    pub count: usize,
364}

366/// Put the results in uppercase and separated with a bullet
367///
368/// # Errors
369///
370/// Returns [`SolverError::MaterializationError`] if the bound entry cannot be found for one of
371/// the `Bindings` objects in `solution`. This indicates an internal solver bug where a solution
372/// was accepted without proper entry bindings.
373pub fn solution_to_string(solution: &[Bindings]) -> Result<String, SolverError> {
374    let str = solution.iter()
375        .enumerate()
376        .map(|(i, b)| {
377            b.get_entry()
378                .ok_or_else(|| SolverError::MaterializationError {
379                    context: format!("Solution missing entry binding at index {i}: {b}")
380                })
381                .map(|c| c.to_ascii_uppercase())
382        })
383        .collect::<Result<Vec<_>, _>>()?
384        .join(" • ");
385
386    Ok(str)
387}

389/// Build a stable key for a full solution (bindings in **pattern order**).
390///
391/// Uses the whole entry binding (`ENTRY_SENTINEL`) to compute the hash.
392///
393/// # Errors
394/// Returns an error if any binding in the solution lacks an entry binding.
395/// This indicates a solver bug where a solution was accepted without entry bindings.
396fn solution_key(solution: &[Bindings]) -> Result<u64, SolverError> {
397    let mut hasher = DefaultHasher::new();
398
399    for (i, b) in solution.iter().enumerate() {
400        let w = b.get_entry().ok_or_else(|| {
401            SolverError::MaterializationError {
402                context: format!("Solution missing entry binding at pattern index {i}: {solution:?}")
403            }
404        })?;
405        w.hash(&mut hasher);
406        // Separator between patterns to avoid ambiguity like ["ab","c"] vs ["a","bc"]
407        HASH_SPLIT.hash(&mut hasher);
408    }
409
410    Ok(hasher.finish())
411}

413/// Simple helper to enforce a wall-clock time limit.
414///
415/// Usage:
416/// ```ignore
417///  let budget = TimeBudget::new(Duration::from_secs(30));
418///  while !budget.expired() {
419///    // do some work
420///  }
421/// ```
422///
423/// You can also query how much time is left (`remaining()`).
424struct TimeBudget {
425    start: Instant,   // when the budget began
426    limit: Duration,  // maximum allowed elapsed time
427}
428
429impl TimeBudget {
430    /// Create a new budget that lasts for `limit` (e.g., 30 seconds).
431    fn new(limit: Duration) -> Self {
432        Self { start: Instant::now(), limit }
433    }
434
435    /// How long this budget has been running.
436    fn elapsed(&self) -> Duration {
437        self.start.elapsed()
438    }
439
440    /// Returns true if the allowed time has fully elapsed.
441    fn expired(&self) -> bool {
442        self.start.elapsed() >= self.limit
443    }
444
445    // Returns the remaining time before expiration, or zero if the budget is already used up.
446    // Unused for now but it may be useful later
447    // fn remaining(&self) -> Duration {self.limit.saturating_sub(self.start.elapsed())}
448}

450macro_rules! timed_stop {
451    // For functions that return Result<(), E>
452    ($budget:expr) => {
453        if $budget.expired() {
454            return Ok(());
455        }
456    };
457    // For functions that return Result<T, E>; caller passes the Ok(...) expr to return
458    ($budget:expr, $ret_expr:expr) => {
459        if $budget.expired() {
460            return Ok($ret_expr);
461        }
462    };
463}

465/// Build a lookup key from a variable set by invoking `fetch` for each variable.
466///
467/// The caller provides a closure that returns the binding value for a given variable
468/// (cloned as `Rc<str>`). Returns `None` if any required key is missing.
469fn build_lookup_key<F>(
470    keys: &HashSet<char>,
471    mut fetch: F,
472) -> Option<LookupKey>
473where
474    F: FnMut(char) -> Option<Rc<str>>,
475{
476    check_invariant!(
477        keys.iter().all(char::is_ascii_uppercase),
478        "All key variables must be one of uppercase A-Z"
479    );
480
481    let mut pairs: Vec<(char, Rc<str>)> = Vec::with_capacity(keys.len());
482    for &var_char in keys {
483        let var_val = fetch(var_char)?;
484        check_invariant!(!var_val.is_empty(), "Variable bindings must be non-empty strings");
485        pairs.push((var_char, var_val));
486    }
487
488    pairs.sort_unstable_by_key(|(c, _)| *c);
489    check_invariant!(
490        pairs.windows(2).all(|w| w[0].0 < w[1].0),
491        "Sorted pairs must be strictly increasing"
492    );
493    Some(pairs)
494}

496/// Build a lookup key from an environment (`HashMap`) given a set of key variables.
497///
498/// Returns `None` if any required key variable is missing in `env`.
499/// Otherwise returns a sorted `LookupKey`.
500fn lookup_key_from_env(
501    env: &HashMap<char, Rc<str>>,
502    keys: &HashSet<char>,
503) -> Option<LookupKey> {
504    build_lookup_key(keys, |var_char| env.get(&var_char).map(Rc::clone))
505}

507/// Build the deterministic lookup key for a binding given the pattern's lookup vars.
508///
509/// Returns `LookupKeyResult::Complete` with a sorted key, or `LookupKeyResult::MissingKey`
510/// if a required variable is not bound.
511///
512/// # Return values
513/// - `Complete(key)` - Successfully built the key (may be empty if `keys` is empty)
514/// - `MissingKey` - A required key variable was not found in the binding
515fn lookup_key_for_binding(
516    binding: &Bindings,
517    keys: &HashSet<char>,
518) -> LookupKeyResult {
519    match build_lookup_key(keys, |var_char| binding.get(var_char).map(Rc::clone)) {
520        Some(pairs) => LookupKeyResult::Complete(pairs),
521        None => LookupKeyResult::MissingKey,
522    }
523}

525/// Push a binding into the appropriate bucket and bump the count.
526fn push_binding(entries: &mut [CandidateBuckets], i: usize, key: LookupKey, binding: Bindings) {
527    check_invariant!(i < entries.len(), "Pattern index {} out of bounds (len={})", i, entries.len());
528    check_invariant!(
529        binding.get_entry().is_some(),
530        "Binding must have an entry set before being pushed"
531    );
532
533    let old_count = entries[i].count;
534    entries[i].buckets.entry(key).or_default().push(binding);
535    entries[i].count += 1;
536
537    debug_assert_eq!(
538        entries[i].count, old_count + 1,
539        "Count must increment by exactly 1"
540    );
541}

543/// Scan a slice of candidate entries against all patterns in the equation context,
544/// materializing any matching bindings into per-pattern buckets.
545///
546/// - Iterates through `entry_list` starting at `start_idx`, up to `batch_size` entries
547///   (or the end of the list).
548/// - For each entry, applies length prefilters and variable constraints from
549///   `equation_context`, pushing any resulting bindings into `entries[i]`.
550/// - May stop early when the [`TimeBudget`] is exhausted; in that case it returns
551///   the number of entries processed so far (and any bindings already pushed remain
552///   in `entries`).
553///
554/// # Arguments
555/// * `entry_list` — master list of candidate entries (scanned by index range)
556/// * `start_idx` — starting index in `entry_list` for this batch
557/// * `batch_size` — maximum number of entries to scan in this batch
558/// * `equation_context` — parsed equation state with patterns, constraints, and hints
559/// * `entries` — mutable slice of per-pattern candidate buckets to be populated
560/// * `budget` — wall-clock time budget to respect
561///
562/// # Returns
563/// The number of entries consumed (≤ `batch_size`). If the time budget expires,
564/// this count reflects the partial progress made before stopping. Callers that
565/// wish to surface a timeout should detect early completion (e.g., by checking
566/// `budget.expired()`) and convert that condition into `SolverError::Timeout`
567/// at a higher level.
568fn scan_batch(
569    entry_list: &[&str],
570    start_idx: usize,
571    batch_size: usize,
572    equation_context: &EquationContext,
573    entries: &mut [CandidateBuckets],
574    budget: &TimeBudget,
575) -> Result<usize, SolverError> {
576    // precondition checks
577    check_invariant!(
578        start_idx <= entry_list.len(),
579        "start_idx ({}) must be <= entry_list.len() ({})",
580        start_idx, entry_list.len()
581    );
582    check_invariant!(
583        entries.len() == equation_context.len(),
584        "entries slice length ({}) must match equation_context pattern count ({})",
585        entries.len(), equation_context.len()
586    );
587    check_invariant!(batch_size > 0, "batch_size must be positive");
588
589    let mut i_entry = start_idx;
590    let end = start_idx.saturating_add(batch_size).min(entry_list.len());
591
592    check_invariant!(
593        end <= entry_list.len(),
594        "end ({}) must be <= entry_list.len() ({})",
595        end, entry_list.len()
596    );
597
598    debug!("Scanning batch: entries [{start_idx}, {end})");
599
600    while i_entry < end {
601        timed_stop!(budget, i_entry);
602
603        let entry = entry_list[i_entry];
604
605        for (i, p) in equation_context.iter().enumerate() {
606            // No per-pattern cap anymore
607
608            // Skip deterministic fully keyed forms
609            if p.is_deterministic && p.all_vars_in_lookup_keys() {
610                continue;
611            }
612            // Cheap length prefilter
613            if !&equation_context.scan_hints[i].is_entry_len_possible(entry.len()) {
614                continue;
615            }
616
617            let matches = match_equation_all(
618                entry,
619                &equation_context.parsed_forms[i],
620                &equation_context.var_constraints,
621                &equation_context.joint_constraints,
622            )?;
623
624            for binding in matches {
625                timed_stop!(budget, i_entry);
626
627                match lookup_key_for_binding(&binding, &p.lookup_keys) {
628                    LookupKeyResult::Complete(key) => {
629                        push_binding(entries, i, key, binding);
630                    }
631                    LookupKeyResult::MissingKey => {
632                        // required key variable not found--skip this binding
633                    }
634                }
635            }
636        }
637
638        i_entry += 1;
639    }
640
641    let scanned = i_entry - start_idx;
642    debug!("Scanned {} entries, populated {} candidate buckets", scanned,
643        entries.iter().map(|b| b.count).sum::<usize>());
644
645    Ok(i_entry)
646}

655/// Depth-first recursive join of per-pattern candidate buckets into full solutions.
656///
657/// This mirrors `recursive_filter` from `umiaq.py`. We walk patterns in order and at each step
658/// select only the bucket of candidates whose shared variables agree with what we've already
659/// chosen (`env`).
660///
661/// Parameters:
662/// - `selected`: the partial solution (one chosen `Bindings` per pattern so far).
663/// - `env`: the accumulated variable → value environment from earlier choices.
664/// - `results`: completed solutions (each is a `Vec<Binding>`, one per pattern).
665/// - `ctx`: join context containing the requested result count, entry set, constraints, and budget.
666/// - `seen`: set of solution hashes to avoid duplicates.
667/// - `rjp`: remaining patterns to process (candidate buckets, lookup keys, pattern info).
668///
669/// Return:
670/// - This function mutates `results` and stops early once `results` contains
671///   `num_results_requested` values.
672/// # Errors
673/// Recursively joins candidate entry bindings to build complete solutions.
674///
675/// # Algorithm Overview
676///
677/// This is a **backtracking search** that explores the space of possible entry assignments
678/// to patterns. It tries to efficiently prune branches using:
679/// 1. **Shared variable bucketing**: candidates are pre-grouped by their shared variable values
680/// 2. **Deterministic fast path**: patterns with all variables already bound are materialized directly
681/// 3. **Early termination**: stops when quota is met or time budget expires
682/// 4. **Deduplication**: uses a hash to avoid returning duplicate solutions
683///
684/// # Details
685///
686/// Given patterns like `AB;BA;BC`, the algorithm:
687/// 1. **Recurse on pattern index**: for pattern `i`, try each candidate binding from its bucket
688/// 2. **Extend environment**: add new variable bindings from the candidate to `env`
689/// 3. **Recurse deeper**: try to match pattern `i+1` with the extended environment
690/// 4. **Backtrack**: restore `env` by removing added variables before trying next candidate
691/// 5. **Base case**: when all patterns matched, check joint constraints and add to `results`
692///
693/// # Fast Path Optimization
694///
695/// For deterministic patterns (e.g., `"cat"` or `AB` when `A` and `B` are already bound):
696/// - materialize the exact entry directly from `env`
697/// - check if it exists in entry list
698/// - skip candidate enumeration entirely
699///
700/// This optimization is critical for performance with long pattern chains.
701///
702/// # Bucketing Example
703///
704/// For equation `AB;BA` with entries `["cat", "dog"]`:
705/// - Pattern 1 `AB`: Candidates bucketed by empty key (no shared vars yet)
706///   - Bucket `[]`: `[{A="c", B="at"}, {A="ca", B="t"}, {A="d", B="og"}, {A="do", B="g"}]`
707/// - Pattern 2 `BA`: Candidates bucketed by `A` (shared variable from pattern 1)
708///   - We scan the entry list again for matches to BA and bucket by value of A:
709///   - Bucket `{A="c"}`: would contain a matches from "atc" (BA = "at"+"c"), but it is not in the entry list
710///   - Bucket `{A="ca"}`: would contain a matches from "tca", but it is not in the entry list
711///   - Bucket `{A="d"}`: would contain a matches from "ogd", but it is not in the entry list
712///   - Bucket `{A="do"}`: would contain a matches from "gdo", but it is not in the entry list
713///
714/// When recursing with, for example, `A="c"` (from "cat" = "c"+"at"), we look up bucket `{A="c"}` for pattern 2.
715/// This bucket is empty (no entries match BA with A="c"), so this branch produces no solutions.
716/// The key optimization: we only check the one bucket, not all 4 bindings from pattern 1.
717///
718/// # Time Complexity
719///
720/// - **Best case**: O(#patterns × b), where b = max bucket size (with good bucketing)
721/// - **Worst case**: O(#candidates ^ #patterns) (with poor bucketing) (i.e., exponential in #patterns)
722///
723/// Pattern ordering (choosing patterns that maximize shared variables) is critical
724/// to keeping bucket sizes small.
725///
726/// # Arguments
727///
728/// - `selected`: stack of bindings chosen so far (one per matched pattern)
729/// - `env`: current variable assignments (e.g., `{A: "cat", B: "dog"}`)
730/// - `results`: accumulator for complete solutions
731/// - `ctx`: context with entry list, constraints, budget, quota
732/// - `seen`: deduplication set (hashes of solutions already found)
733/// - `rjp`: remaining patterns to match (slice shrinks with each recursion level)
734///
735/// # Returns
736///
737/// `Ok(())` when search completes (quota met or exhausted candidates).
738///
739/// Returns `Err(SolverError::Timeout)` if the time budget expires during the join.
740/// In that case, partial solutions already found remain in `results`.
741fn recursive_join(
742    selected: &mut Vec<Bindings>,
743    env: &mut HashMap<char, Rc<str>>,
744    results: &mut Vec<Vec<Bindings>>,
745    ctx: &JoinCtx,
746    seen: &mut HashSet<u64>,
747    rjp: &[RecursiveJoinParameters],
748) -> Result<(), SolverError> {
749    recursive_join_inner(
750        selected,
751        env,
752        results,
753        ctx,
754        seen,
755        rjp,
756        rjp.len(),
757    )
758}

760fn recursive_join_inner(
761    selected: &mut Vec<Bindings>,
762    env: &mut HashMap<char, Rc<str>>,
763    results: &mut Vec<Vec<Bindings>>,
764    ctx: &JoinCtx,
765    seen: &mut HashSet<u64>,
766    rjp: &[RecursiveJoinParameters],
767    total_patterns: usize, // for debug assert
768) -> Result<(), SolverError> {
769    // Invariant: selected.len() + rjp.len() == total_patterns
770    // (we've processed 'selected' patterns and have 'rjp' remaining)
771    debug_assert_eq!(
772        selected.len() + rjp.len(), total_patterns,
773        "selected ({}) + remaining ({}) must equal total patterns ({})",
774        selected.len(), rjp.len(), total_patterns
775    );
776
777    // Stop if we've met the requested quota of full solutions.
778    if results.len() >= ctx.num_results_requested {
779        return Ok(());
780    }
781
782    timed_stop!(ctx.budget);
783
784    if let Some(rjp_cur) = rjp.first() {
785        // ---- FAST PATH: deterministic + fully keyed ----------------------------
786        let p = &rjp_cur.pattern;
787        if p.is_deterministic && p.all_vars_in_lookup_keys() {
788            // The entry is fully determined by literals + already-bound vars in `env`.
789            let Some(expected) = rjp_cur.parsed_form
790                .materialize_deterministic_with_env(env)
791            else {
792                return Err(SolverError::MaterializationError {
793                    context: format!(
794                        "failed to materialize deterministic pattern with variables: {:?}\n  \
795                        environment: {:?}\n  \
796                        pattern depth: {}/{}",
797                        p.variables,
798                        env,
799                        selected.len() + 1,
800                        total_patterns
801                    )
802                });
803            };
804
805            if !ctx.entry_set.contains(expected.as_str()) {
806                // This branch cannot succeed — prune immediately.
807                return Ok(());
808            }
809
810            // Build a minimal Bindings for this pattern:
811            // - include ENTRY_SENTINEL (whole entry)
812            // - include only vars that belong to this pattern (they must already be in env)
813            let mut binding = Bindings::default();
814            binding.set_entry(&expected);
815            for &var_char in &p.variables {
816                // safe: all vars in lookup_keys must be in env by construction
817                if let Some(var_val) = env.get(&var_char) {
818                    binding.set_rc(var_char, Rc::clone(var_val));
819                } else {
820                    // this should never happen--indicates a logic error in solver
821                    check_invariant!(
822                        false,
823                        "Variable '{}' in lookup_keys but not in env--solver invariant violated",
824                        var_char
825                    );
826                }
827            }
828
829            selected.push(binding);
830            recursive_join_inner(selected, env, results, ctx, seen, &rjp[1..], total_patterns)?;
831            selected.pop();
832            return Ok(()); // IMPORTANT: skip normal enumeration path
833        }
834        // ------------------------------------------------------------------------
835
836        // Decide which bucket of candidates to iterate for pattern `idx`.
837        //
838        // We must create the deterministic key
839        //   `Some(sorted_pairs)` using the current `env` and fetch that bucket.
840        //   (This includes the case keys.is_empty() → key is `Some([])`.)
841        let bucket_candidates_opt: Option<&Vec<Bindings>> = {
842            timed_stop!(ctx.budget);
843            // Build lookup key from current environment
844            let Some(key) = lookup_key_from_env(env, rjp_cur.lookup_keys) else {
845                // If any required var isn't bound yet, there can be no matches for this branch.
846                return Ok(());
847            };
848
849            rjp_cur.candidate_buckets.buckets.get(&key)
850        };
851
852        // If there are no candidates in that bucket, dead-end this branch.
853        let Some(bucket_candidates) = bucket_candidates_opt else {
854            return Ok(());
855        };
856
857        // Try each candidate binding for this pattern.
858        for cand in bucket_candidates {
859
860            timed_stop!(ctx.budget);
861
862            if results.len() >= ctx.num_results_requested {
863                break; // stop early if we've already met the quota
864            }
865
866            // Defensive compatibility check: if a variable is already in `env`,
867            // its value must match the candidate. This *should* already be true
868            // because we selected the bucket using the shared vars—but keep this
869            // in case upstream bucketing logic ever changes.
870            if cand.iter().filter(|(var_char, _)| *var_char != ENTRY_SENTINEL).any(|(var_char, var_val)| env.get(&var_char).is_some_and(|prev| prev != var_val)) {
871                continue;
872            }
873
874            // Extend `env` with any *new* bindings from this candidate (don't overwrite).
875            // Track what we added so we can backtrack cleanly.
876            let mut added_vars: Vec<char> = Vec::with_capacity(10); // typically few vars per pattern
877            for (var_char, var_val) in cand.iter() {
878                if var_char == ENTRY_SENTINEL {
879                    continue;
880                }
881                if let Entry::Vacant(e) = env.entry(var_char) {
882                    e.insert(Rc::clone(var_val));
883                    added_vars.push(var_char);
884                }
885            }
886
887            // Choose this candidate for pattern `idx` and recurse for `idx + 1`.
888            selected.push(cand.clone());
889            recursive_join_inner(selected, env, results, ctx, seen, &rjp[1..], total_patterns)?;
890            selected.pop();
891
892            // Backtrack: remove only what we added at this level.
893            for k in added_vars {
894                env.remove(&k);
895            }
896        }
897    } else {
898        // Base case: if we've placed all patterns, `selected` is a full solution.
899        if ctx.joint_constraints.all_strictly_satisfied_for_parts(selected)
900            && ctx.var_constraints.check_not_equal(selected)
901            && seen.insert(solution_key(selected)?) {
902            debug!("Found solution #{}: {} patterns matched", results.len() + 1, selected.len());
903            results.push(selected.clone());
904        }
905    }
906
907    Ok(())
908}

910/// Read in an equation string and return results from the entry list
911///
912/// This orchestrates parsing the input string into an [`EquationContext`],
913/// scanning candidate entries in adaptive batches, and recursively joining
914/// variable bindings into full solutions. Results are accumulated until either
915/// the requested number of solutions is found or the [`TimeBudget`] expires.
916///
917/// # Arguments
918/// - `input`: equation string to parse (e.g., `"AB;BC;CA"`).
919/// - `entry_list`: slice of candidate entries to match against.
920/// - `num_results_requested`: optional cap on how many solutions to return
921///   (use `None` to search exhaustively).
922///
923/// # Returns
924/// A vector of fully materialized [`Solution`]s, up to the requested limit.
925///
926/// # Errors
927/// Returns a [`SolverError`] if:
928/// - the equation string cannot be parsed (`ParseFailure`),
929/// - a binding cannot be materialized into a valid solution (`MaterializationError`),
930/// - or the solver exceeds its wall-clock time budget (`Timeout`).
931///
932/// On timeout, any partial solutions discovered before expiration are discarded
933/// and the error is bubbled up to the caller, so the end user sees an explicit
934/// failure rather than a silently truncated result set.
935pub fn solve_equation(
936    input: &str,
937    entry_list: &[&str],
938    num_results_requested: usize,
939) -> Result<SolveResult, SolverError> {
940    solve_equation_with_budget(input, entry_list, num_results_requested, Duration::from_secs(TIME_BUDGET))
941}

943/// Internal solver with custom time budget (used for testing).
944#[cfg_attr(not(test), allow(dead_code))]
945fn solve_equation_with_budget(
946    input: &str,
947    entry_list: &[&str],
948    num_results_requested: usize,
949    time_budget: Duration,
950) -> Result<SolveResult, SolverError> {
951    // Precondition: validate input at API boundary
952    if input.is_empty() {
953        return Err(SolverError::ParseFailure(Box::new(ParseError::EmptyForm)));
954    }
955    check_invariant!(
956        num_results_requested > 0,
957        "num_results_requested must be positive"
958    );
959    check_invariant!(
960        entry_list.iter().all(|w| !w.is_empty()),
961        "All entries in entry list must be non-empty"
962    );
963
964    // 1. Make a hash set version of our entry list
965    let entry_list_as_set = entry_list.iter().copied().collect();
966
967    // 2. Parse the input equation string into our `EquationContext` struct.
968    //    This holds each pattern string, its parsed form, and its `lookup_keys` (shared vars).
969    let equation_context = match input.parse::<EquationContext>() {
970        Ok(ctx) => ctx,
971        Err(e) => {
972            error!("Failed to parse equation '{input}': {e}");
973            return Err(e.into());
974        }
975    };
976
977    info!("Parsed equation with {} patterns: {}", equation_context.len(), input);
978    debug!("{equation_context}");
979
980    // If there are no patterns, propagate a NoPatterns error.
981    if equation_context.len() == 0 {
982        return Err(SolverError::NoPatterns);
983    }
984
985    // Warn about potentially expensive queries
986    if equation_context.len() >= PATTERN_COUNT_WARNING_THRESHOLD {
987        warn!("Solving {} patterns may be computationally expensive", equation_context.len());
988    }
989
990    // 3. Prepare storage for candidate buckets, one per pattern.
991    //    `CandidateBuckets` tracks (a) the bindings bucketed by shared variable values, and
992    //    (b) a count so we can stop early if a pattern gets too many matches.
993    // Mutable because we fill buckets/counts during the scan phase.
994    let mut entries: Vec<CandidateBuckets> = Vec::with_capacity(equation_context.len());
995    for _ in &equation_context {
996        entries.push(CandidateBuckets::default());
997    }
998
999    // 4. Pull out some data from equation_context
1000    let lookup_keys = &equation_context.lookup_keys;
1001    let parsed_forms = &equation_context.parsed_forms;
1002    let joint_constraints = &equation_context.joint_constraints;
1003
1004    // 5. Iterate through every candidate entry.
1005    let budget = TimeBudget::new(time_budget);
1006
1007    let mut results: Vec<Vec<Bindings>> = Vec::with_capacity(num_results_requested.min(1000));
1008    let mut selected: Vec<Bindings> = Vec::with_capacity(equation_context.len());
1009    let mut env: HashMap<char, Rc<str>> = HashMap::with_capacity(26); // max 26 variables A-Z
1010
1011    // scan_pos tracks how far into the entry list we've scanned.
1012    let mut scan_pos = 0;
1013
1014    // Global set of fingerprints for already-emitted solutions.
1015    // Ensures we don't return duplicate solutions across scan/join rounds.
1016    let mut seen: HashSet<u64> = HashSet::with_capacity(num_results_requested.min(1000));
1017
1018    // batch_size controls how many entries to scan this round (adaptive).
1019    let mut batch_size = DEFAULT_BATCH_SIZE;
1020
1021    // High-level solver driver. Alternates between:
1022    //   1. scanning more entries from the dictionary into candidate buckets
1023    //   2. recursively joining those buckets into full solutions
1024    // Continues until either we have enough results, the entry list is exhausted,
1025    // or the time budget expires.
1026    #[cfg(debug_assertions)]
1027    let mut iteration = 0;
1028
1029    // Track whether we've already issued certain warnings (warn once)
1030    let mut warned_about_large_candidates = false;
1031
1032    while results.len() < num_results_requested
1033        && scan_pos < entry_list.len()
1034        && !budget.expired()
1035    {
1036        #[cfg(debug_assertions)]
1037        {
1038            iteration += 1;
1039            if iteration % 10 == 0 {
1040                debug!(
1041                    "Solver iteration {}: scan_pos={}/{}, batch_size={}, results={}/{}, elapsed={:.2}s",
1042                    iteration, scan_pos, entry_list.len(), batch_size,
1043                    results.len(), num_results_requested,
1044                    budget.elapsed().as_secs_f64()
1045                );
1046            }
1047        }
1048
1049        // 1. Scan the next batch_size entries into candidate buckets.
1050        // Each candidate binding is grouped by its lookup key so later joins are fast.
1051        let new_pos = match scan_batch(
1052            entry_list,
1053            scan_pos,
1054            batch_size,
1055            &equation_context,
1056            &mut entries,
1057            &budget,
1058        ) {
1059            Ok(pos) => pos,
1060            Err(e) => {
1061                error!("Error during entry list scanning at position {scan_pos}: {e}");
1062                return Err(e);
1063            }
1064        };
1065        scan_pos = new_pos;
1066
1067        // Warn if candidate buckets are growing very large
1068        if !warned_about_large_candidates {
1069            let total_candidates: usize = entries.iter().map(|b| b.count).sum();
1070            if total_candidates >= LARGE_CANDIDATE_COUNT_THRESHOLD {
1071                warn!("Large candidate count ({total_candidates} bindings)--search space may be very large");
1072                warned_about_large_candidates = true;
1073            }
1074        }
1075
1076        if budget.expired() {
1077            break;
1078        }
1079
1080        // 2. Attempt to build full solutions from the candidates accumulated so far.
1081        // This may rediscover old partials, so we use `seen` at the base case
1082        // to ensure only truly new solutions are added to `results`.
1083        let rjp = entries.iter().zip(lookup_keys.iter()).zip(equation_context.ordered_list.iter()).zip(parsed_forms.iter())
1084            .map(|(((candidate_buckets, lookup_keys), p), parsed_form)| {
1085                RecursiveJoinParameters {
1086                    candidate_buckets,
1087                    lookup_keys,
1088                    pattern: p,
1089                    parsed_form,
1090                }
1091            }).collect::<Vec<_>>();
1092
1093        // Set up the context for `recursive_join`
1094        let ctx = JoinCtx {
1095            num_results_requested,
1096            entry_set: &entry_list_as_set,
1097            joint_constraints,
1098            var_constraints: &equation_context.var_constraints,
1099            budget: &budget,
1100        };
1101
1102        // Call `recursive_join`
1103        debug!("Starting recursive join with {} candidate buckets (currently {} solutions)",
1104            entries.len(), results.len());
1105        if let Err(e) = recursive_join(
1106            &mut selected,
1107            &mut env,
1108            &mut results,
1109            &ctx,
1110            &mut seen,
1111            &rjp,
1112        ) {
1113            error!("Error during recursive join: {e}");
1114            return Err(e);
1115        }
1116
1117        // We exit early in two cases
1118        // 1. We've hit the number of results requested
1119        // 2. We have no more entries to scan
1120        if results.len() >= num_results_requested ||
1121            scan_pos >= entry_list.len() {
1122            break;
1123        }
1124
1125        if budget.expired() {
1126            break;
1127        }
1128
1129        // Grow the batch size for the next round
1130        // TODO maybe adaptive resizing?
1131        let old_batch_size = batch_size;
1132        batch_size = batch_size.saturating_mul(BATCH_SIZE_MULTIPLIER);
1133        if batch_size != old_batch_size {
1134            debug!("Increasing batch size: {old_batch_size} → {batch_size}");
1135        }
1136    }
1137
1138    // Determine status before consuming results
1139    let status = if budget.expired() {
1140        warn!("Solver timed out after {:.2}s with {} results", budget.elapsed().as_secs_f64(), results.len());
1141        SolveStatus::TimedOut { elapsed: budget.elapsed() }
1142    } else if results.len() >= num_results_requested {
1143        info!("Found {} results (requested: {})", results.len(), num_results_requested);
1144        SolveStatus::FoundEnough
1145    } else {
1146        info!("Exhausted entry list with {} results", results.len());
1147        SolveStatus::EntryListExhausted
1148    };
1149
1150    // ---- Reorder solutions back to original form order ----
1151    let reordered = results.into_iter().map(|mut solution| {
1152        let mut reordered_solution = Vec::with_capacity(solution.len());
1153        for original_i in 0..solution.len() {
1154            let ordered_i = equation_context.original_to_ordered[original_i];
1155            // Move elements by swapping with default values
1156            reordered_solution.push(std::mem::take(&mut solution[ordered_i]));
1157        }
1158        reordered_solution
1159    }).collect::<Vec<_>>();
1160
1161    // Postcondition: verify solution structure is consistent
1162    debug_assert!(
1163        reordered.len() <= num_results_requested,
1164        "Number of solutions ({}) must not exceed requested ({})",
1165        reordered.len(), num_results_requested
1166    );
1167    debug_assert!(
1168        reordered.iter().all(|sol| sol.len() == equation_context.len()),
1169        "Each solution must have exactly {} bindings (one per pattern)",
1170        equation_context.len()
1171    );
1172    debug_assert!(
1173        reordered.iter().all(|sol| sol.iter().all(|b| b.get_entry().is_some())),
1174        "All bindings in solutions must have an entry set"
1175    );
1176
1177    Ok(SolveResult { solutions: reordered, status, readable_equation_context: equation_context.readable_context() })
1178}
